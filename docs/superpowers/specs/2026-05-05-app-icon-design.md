# App icon and favicon — design spec

**Date**: 2026-05-05
**Status**: design approved, awaiting implementation plan

## Goal

Replace the placeholder Vite favicon and ship a polished, platform-correct
app icon for web (favicon), iOS (home-screen icon), and Android (adaptive
icon). Single SVG source feeds every output via `@capacitor/assets`.

## The design

A tilted Ace of Diamonds card sitting on a felt-green vignette — option **P4**
from brainstorm. The card has a soft drop shadow, the felt has a subtle
radial vignette, and the card carries the corner "A♦" indexes plus a large
centered diamond pip.

### Specifications

| Element            | Value                                       |
| ------------------ | ------------------------------------------- |
| Canvas (master)    | 1024 × 1024 px (SVG `viewBox="0 0 200 200"` source) |
| Background         | Radial gradient, center `#1c4a2a` → mid `#0f3818` (60%) → edge `#072010` |
| Card body          | `#fffaf0` cream white                       |
| Card corner radius | 9 / 200 of canvas (~4.5%)                   |
| Card dimensions    | 90 × 135 / 200 (45% × 67.5% of canvas)      |
| Card rotation      | −7° around canvas center                    |
| Card drop shadow   | dx 2, dy 5, blur 3, opacity 0.45            |
| Suit / accents     | `#c41e3a` classic playing-card red          |
| Pip glyph          | `♦` Georgia serif, font-size 64 / 200 (~32%) |
| Corner indexes     | "A" + "♦" mirrored top-left and bottom-right, font-size 22 / 200 |
| Outer mask radius  | 40 / 200 (~20%, iOS-style rounded square)   |

### Composition (200 × 200 viewBox)

```
- rect 0,0 200×200, fill: feltGrad (radial)
- group rotate(-7° around 100,100):
    - rect 55,32 90×135 rx=9 fill=#fffaf0 + dropShadow
    - text "A"  64,58  size 22 weight 700 fill=#c41e3a
    - text "♦"  64,80  size 22                  fill=#c41e3a
    - group rotate(180° around 100, 99.5):
        - text "A"  64,58  ...
        - text "♦"  64,80  ...
    - text "♦" 100,135  size 64 text-anchor=middle fill=#c41e3a
```

## Platform variants

The same master design is rendered three different ways:

### Web favicon (`public/favicon.svg` + PNG fallbacks)

- Full-bleed master, served as SVG so it scales crisply.
- PNG fallbacks at 32 × 32 and 16 × 16 for older browsers.
- Linked from `index.html` via `<link rel="icon" type="image/svg+xml">` plus
  legacy PNG `<link>`.

### iOS app icon (`resources/icon.png`)

- Full-bleed master at 1024 × 1024. iOS applies its own rounded-square mask
  on the home screen, so the master's rounded corners are decorative — they
  get replaced.
- `@capacitor/assets` generates the full Apple icon set
  (`AppIcon-{size}.png`) into `ios/App/App/Assets.xcassets/AppIcon.appiconset/`.

### Android adaptive icon (`resources/icon-foreground.png` + `icon-background.png`)

Android masks the foreground separately from the background and may apply
circle, squircle, or rounded-square shapes. Splitting the layers prevents
the card from being clipped:

- **Background layer** — felt green with vignette, full bleed. No card.
- **Foreground layer** — card only, on transparent background, scaled to
  78% (linear, applied to width and height) so it sits inside the safe
  zone (the inscribed circle of the canvas) and survives every OEM mask.

`@capacitor/assets` generates the legacy `mipmap-{density}/ic_launcher.png`
plus the adaptive XML and density-bucketed foreground/background PNGs.

### Splash screen (`resources/splash.png`)

- 2732 × 2732 felt-green background with the master icon centered at ~33%
  of the canvas — small enough that it doesn't crowd phones with notches.
- `@capacitor/assets` derives every iOS launch image and Android splash
  density from this single source.

## Generation pipeline

Source of truth is SVG (hand-editable, version-friendly).
`@capacitor/assets` expects PNG input, so rasterize SVG → PNG before
running the generator. One npm script chains both steps:

```bash
npm install --save-dev @capacitor/assets sharp @resvg/resvg-js
npm run assets:generate
```

Under the hood, `assets:generate` runs `scripts/build-icons.mjs` which:

1. Rasterizes `resources/*.svg` to matching PNG paths
   (`icon.png` 1024², `icon-foreground.png` 1024², `icon-background.png`
   1024², `splash.png` 2732²) using `@resvg/resvg-js`.
2. Renders `resources/icon.svg` to `public/favicon-32.png` and
   `public/favicon-16.png`.
3. Copies `resources/icon.svg` to `public/favicon.svg`.
4. Invokes `capacitor-assets generate` to populate native projects.

The Node version + JDK preflight isn't needed for the rasterization step
itself, but `capacitor-assets generate` writes into `ios/` and `android/`,
so the script reuses the env detection from `scripts/run-android.sh`.

## Acceptance criteria

- The browser tab favicon shows the Ace of Diamonds at all zoom levels
  (16 px through 64 px). No fuzzy upscaling, no Vite default.
- iOS home-screen icon: full-bleed felt with the tilted card; no white
  border, no transparent corners.
- Android home-screen icon: card stays fully visible under circle, squircle,
  and rounded-square masks. Felt fills the entire icon area regardless of
  mask shape (no transparent edges showing through).
- Splash screen displays the icon centered on the felt-green background
  during the 800 ms launch window already configured in
  `capacitor.config.ts`.
- Re-running `npm run assets:generate` after a clean re-scaffold of `ios/`
  or `android/` reproduces the assets without manual steps.

## Out of scope

- Animated launch / Lottie splash. The static splash matches the rest of
  the app's restraint.
- Notification icons (no push notifications shipped yet).
- Dark-mode favicon variant. The felt green works on both light and dark
  browser themes.
- Apple Touch Icon for "Add to Home Screen" on the web version. Capacitor
  handles the native iOS install path; PWA install is not a goal.
